# Skill: finish-dev

## 目的
開発作業を完了し、成果物を整理・反映する。

## 手順
1. ドキュメント類（`docs/` 配下）が最新化されていることを確認する。
    - `backlog.md` には未完了のタスクのみを残す。
2. 今回の開発で実施した内容のサマリーを報告し、ユーザーから完了の承認を得る。
3. [STOP!] ユーザーの承認を得るまで次のステップに進まない。
4. 承認されたら、コミットおよびマージを実行する。
