# Skill: finish-dev

## 目的
開発作業を完了し、成果物を整理・反映する。

## 手順
1. ドキュメント類（`docs/` 配下）が最新化されていることを確認する。
    - `backlog.md` には完了した項目は残さず、未完了のタスクのみを記載する。
    - `update_history.json` を作成または更新する。記載ルールは以下の通り：
        - βバージョン(v0.x)の期間は記載しない。
        - バージョン表記はマイナーまでとし、パッチバージョンは含めない（"X.X" 形式）。
        - 記載内容はゲームプレイヤーにとってのメリット（ユーザー体験の向上）を中心に記述する。
        - 開発者向けのテクニカルな内容（リファクタリング、テスト実施報告など）は記載しない。
2. 今回の開発で実施した内容のサマリーを報告し、ユーザーから完了の承認を得る。
3. [STOP!] ユーザーの承認を得るまで次のステップに進まない。
4. 承認されたら、コミットおよびマージを実行する。
