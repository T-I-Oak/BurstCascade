# BURST CASCADE 仕様書

## 1. 基本情報
- **アプリ名**: BURST CASCADE (バースト・カスケード)
- **ジャンル**: 連鎖爆発ストラテジー
- **先攻決定ルール**:
  ゲーム開始時に「エネルギーバースト」演出が発生し、先攻プレイヤーがランダムに抽選される。抽選結果は、エネルギー粒子が吸い込まれた側のプレイヤーラベルが強烈に発光することで示される。
- **引き分け（Draw）判定**:
  同一の連鎖アクション内ですべての勢力の「共鳴中枢（Core）」が同時に消失した場合、引き分け（DRAW - RESONANCE VOID）としてゲームが終了する。
- **概要**:
  純粋なエネルギーの結晶体「ヘキサグリッド」を舞台に、エネルギーの注入と爆発（Burst）、そして連鎖（Cascade）を駆使して、対立する勢力の「共鳴中枢（Core）」を奪還・無効化する戦略ゲーム。

## 2. システム構成

### 2.1. レンダリング・座標系 (Unified Coordinate System)
本作は多様なデバイス環境で一貫した操作感と視覚体験を提供するため、**論理座標系**と**物理座標系**を厳密に分離している。

- **High-DPI (DPR) 対応**:
  - `devicePixelRatio` に基づき Canvas の内部解像度を物理ピクセルに合わせつつ、描画コンテキスト自体に `scale(dpr, dpr)` を適用している。
  - **論理座標（CSSピクセル）**: マップレイアウト計算、当たり判定（`findHexAt`）、パーティクル座標計算、および選択状態管理（`selectedHex`）に使用。
  - **物理座標（DPR補正）**: キャンバス内部解像度、フォントサイズの決定に使用。
- **PointerEvent 体系の採用**:
  - マウスとタッチを確実に判別し、300ms のクリック遅延を排除するため、`click` ではなく `pointerup` を主操作イベントとして使用。
  - **ブラウザジェスチャーの抑制**: キャンバスに `touch-action: none` を適用し、かつ `touchmove` リスナーに `{ passive: false }` を明示設定することで、標準的なスクロールやズーム干渉を完全に排除。
  - **動的選択と確定の分離**: `isTouchMoved` フラグを導入。タッチスライド（`touchmove`）中はそのマスを「選択（Hover）」対象とするが、指を離した瞬間の `pointerup` では常に「選択（Select）」状態を確定させるにとどめ、実際の「注入（Confirm）」はそのマスへの次回の単独タップによってのみ実行されるように分離。これにより、スライド操作直後の意図しない誤爆を防止。
  - **ハイライト表示の一貫性**: 共通メソッド `updateHoveredNeighbors` により、スライド操作中とタップ操作（選択時）の両方で、周囲マスのプレビュー表示が正しく更新・維持されるように改善。
  - **デバイス別操作最適化**: マウス操作（PC）では 1 クリック即時確定、タッチ操作（モバイル）では 2 タップ確定（またはスライド後に 1 タップ確定）を明確に分離し、各プラットフォームでの最高のエクスペリエンスを保証。
- **レイアウトの安定化**:
  - iPad / モバイルブラウザでの表示崩れを防ぐため、オーバーレイ要素を `position: absolute` で `main` 要素内に配置し、ヘッダーとのスタッキングコンテキストの干渉を回避している。

### 2.2. パーティクル・演出システム (Dynamic FX)
- **3D 垂直座標補正**:
  - ヘクスが積み重なって厚みを持つため、パーティクル生成時には地面（y=0）ではなく、各ヘクス現在の高さ（天面）を考慮した `y - height` の座標を起点とする。
- **演出の適正化**: 物理的な衝撃が伴わないアクション（高さ0のタイル配置等）ではパーティクル生成を抑止し、視覚的な一貫性を保つ。
- **カスケード演出**: 連鎖爆発、パワー再構築ドット、コアダメージ粒子など、全ての演出が論理座標ベースで同期して描画される。

### 2.3. オーディオシステム (Dynamic Rhythmic Engine)
本作のオーディオは、Web Audio API による多層的な「状況連動型ダイナミック・シンセシス」を採用している。

- **ブラウザ・ポリシー準拠**:
  - `AudioContext` のアクティベーションは、正規のユーザージェスチャ（`click`, `touchend`, `keydown`, `pointerup`）時にのみ実行される。
  - **非同期レジリエンス起動**:
    - 起動プロセス（`resume()`）を `async/await` で管理し、完了を確実に待機。さらに実行中ロック（`isActivating`）を導入し、多重起動エラーを防止。
    - リスナーから `touchstart` を除外。ブラウザがジェスチャーと認めない可能性があるイベントを排除し、コンソールエラーを抑制。
    - リスナーの `passive` 設定を無効化し、モバイルブラウザでのジェスチャー認識精度を最大化している。
  - **サイレント・アンロック**: 初動ジェスチャ時に無音バッファを再生し、モバイル等の自動再生制限を確実に解除する。
- **BGM ロジック**:
  - **ダイナミック・シンセシス**: まろやかな音色（P1：影の木琴調、P2：高揚感のあるベル調）をリアルタイムに合成・生成。
  - **状況連動型スケーリング (Intensity)**: 各勢力の残りコア数が減るに従い、楽曲のインテンシティ（音階・密度）が段階的に上昇し、聴覚的な緊張感を高める。
  - **ピンチ状態 (Pinch)**: いずれかのプレイヤーのコアが残り1つになると、BPM が上昇し、警報音と重いドローンが加わる緊急状態へ遷移する。
  - **勝敗演出 (Drum Finale)**: 
    - 勝利・敗北に関わらず、共通の重厚なドラムフィニッシュ（キック、スネア、ノイズ、重低音の複合演出）が再生される。楽曲の締めくくりのようなタイトな響きでゲームを完結させる。
- **FX チェーン**: コンプレッサー、リバーブ、ディレイを多段接続し、ブラウザ上でのリッチな音響空間を実現。

### 2.4. データ管理 (Data Persistence)
- **DataManager**: `DataManager` クラスが LocalStorage の読み書きと、アプリのメジャーバージョン（`__APP_VERSION__`）に応じた自動マイグレーション処理を一元管理します。

## 3. 機能一覧
- **PLAYERS**: 1P vs CPU (PvC) / 2P LOCAL (PvP)
- **AI LEVEL**: EASY / NORMAL / HARD
- **MAP SIZE**: MINI / REGULAR
- **実績システム**: 難易度・マップ別の進捗管理
- **チュートリアル**: 初回プレイ時やヘルプボタンからの演出ガイド

---

## 4. 技術スタック
- **Core**: HTML5, CSS3 (Vanilla), Vanilla JavaScript
- **Module System**: ES Modules (ESM)
- **Build Tool**: Vite
- **Sound**: Web Audio API (Manual Synthesis & Scheduling)
- **Testing**: Vitest (jsdom environment)

## 6. ビルド・開発環境
- **Vite**: 開発サーバーおよびプロダクションビルドに使用。
- **Vitest**: ユニットテストフレームワーク。`npm test` で全てのユニットテストを実行します。
- **バージョン管理**: `package.json` の `version` フィールドおよび `update_history.json` で管理されます。
