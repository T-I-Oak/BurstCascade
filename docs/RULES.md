# プロジェクト開発ルール

## ドキュメント作成ルール
- **開発計画書**: `docs/dev_implementation_plan.md` として作成する。
- **引継ぎ資料**: `docs/dev_handover.md` として作成する。
- **バックログ**: `backlog.md` を使用し、**完了した項目は残さない**（完了次第削除する）。

## 更新履歴 (`update_history.md`) の記載ルール
プレイヤー（ユーザー）に向けた情報を中心に記載し、開発者目線の情報は排除する。

- **対象バージョン**:
    - βバージョン (v0.x) の期間は記載しない。
    - バージョン表記はマイナーまでとし、パッチバージョンは記載しない（例: `1.0`, `1.1`）。
- **記載内容**:
    - ユーザーに与えるメリットを中心に記載する。
    - **ノイズとなる情報の除外**: リファクタリング、内部ロジックの変更、テストの実施内容などは記載しない。
    - **読者ターゲット**: ゲームプレイヤー。専門用語を避け、どのような体験が変わったかを記述する。
