# オーディオシステム

## 1. 概要 (Dynamic Rhythmic Engine)
Web Audio API による多層的な「状況連動型ダイナミック・シンセシス」を採用。

### 1.1. ブラウザ・ポリシー準拠
- `AudioContext` のアクティベーションは、正規のユーザージェスチャ（`click`, `touchend`, `keydown`, `pointerup`）時にのみ実行。
- iPadOS で UI の click 処理より先に音声解除を行えるよう、初回ジェスチャは capture フェーズで検出する。
- **非同期レジリエンス起動**: 起動プロセス（`resume()`）を `async/await` で管理。実行中ロック（`isActivating`）を導入し、多重起動エラーを防止。
- **サイレント・アンロック**: 初動ジェスチャ時に無音バッファを再生し、モバイル等の自動再生制限を確実に解除。
- **保留BGMの再開**: iPadOS 等で `AudioContext` が一度 `suspended` になった場合でも、要求された BGM パターンを保持し、ユーザージェスチャによる `resume()` 成功後に再開する。
- iPad Safari では、最初のジェスチャで `AudioContext` を作成し、同じタップ内の後続イベントまたは次のジェスチャで `resume()` を再試行する 2 段階起動を行う。
- ページ復帰時は `visibilitychange` / `pageshow` / `focus` で `AudioContext.resume()` とBGMスケジューラ再開を試行する。

## 2. BGM ロジック
- **ダイナミック・シンセシス**: まろやかな音色（P1：影の木琴調、P2：高揚感のあるベル調）をリアルタイムに合成生成。
- **状況連動型スケーリング (Intensity)**: 各勢力の残りコア数が減るに従い、楽曲の音階・密度が段階的に上昇し、緊張感を高める。
- **ピンチ状態 (Pinch)**: いずれかのプレイヤーのコアが残り1つになると、BPM が上昇し、警報音と重いドローンが加わる緊急状態へ遷移。
- **勝敗演出 (Drum Finale)**: 勝利・敗北に関わらず、共通の重厚なドラムフィニッシュ（キック、スネア、ノイズ、重低音の複合演出）が再生され、ゲームを完結させる。

## 3. FX チェーン
- コンプレッサー、リバーブ、ディレイを多段接続し、ブラウザ上でのリッチな音響空間を実現。
