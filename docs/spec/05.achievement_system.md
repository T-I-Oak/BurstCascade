# 実績システム (Achievement System)

## 1. 概要
実績システムは、プレイヤーが特定の戦略や目標を達成した際に解除されるやりこみ要素です。難易度（EASY / NORMAL / HARD）およびマップサイズ（MINI / REGULAR）の全6パターンの組み合わせで進捗を独立して追跡・記録します。

## 2. 実績一覧と判定論理仕様

実績の判定はすべて**ゲーム終了時（勝敗確定時）**に行われます。

## 3. UI/UX・デザイン仕様

### 3.1. ダッシュボード（カードグリッドUI）
- 各実績はカード形式でグリッド状に配置されます。
- **カードのレイアウト**: 2カラム構成。
  - **左側**: 実績のタイトルと説明文。
  - **右側**: `MINI` と `REGULAR` のそれぞれの難易度（`EASY` / `NORMAL` / `HARD`）に対応するメダルスロット（計6個）のマトリックス表示。
- **公開条件 (Revealed)**:
  - プレイヤーが一度でも解除した実績、または未公開リストの上位2件までのみ詳細が表示されます。未解除のロックされた実績は「???」と表示されます。

### 3.2. 進捗に応じた段階的ビジュアル変化
1. **未獲得 (0/6)**: カード左側が暗い背景、スロットは空でグレー文字の「EASY」「NORM」「HARD」のラベルを表示。
2. **一部獲得 (1〜5/6)**: カードの左側がゴールド（グラデーション）になり、獲得した難易度のスロットにメダル（SVG）がポップアップ演出とともに表示されます。
3. **完全制覇 (6/6 - MASTERED)**: カードの枠線が発光し、カードの右側を含めた全面がゴールドになり、プレミアムな輝き（ボックスシャドウ）を放つ演出が適用されます。

### 3.3. 実績メダルのビジュアル差別化
難易度とマップサイズによって、メダル (SVG) の表現を差別化します。
- **難易度別カラー (メダルベース)**:
  - **EASY**: 銅（Bronze - `#cd7f32`）
  - **NORMAL**: 銀（Silver - `#c0c0c0`）
  - **HARD**: 金（Gold - `#ffd700`）
- **マップサイズ別の差別化 (メダルリボン)**:
  - **REGULAR**: 赤いリボンの真ん中に「縦の青いストライプ（`#3b82f6`）」を描画し、より豪華で立体的な勲章デザインにします。
  - **MINI**: リボンは通常の赤単色とし、メダル全体の彩度（`filter: saturate(0.6)`）や透明度（`opacity: 0.85`）を若干落とし、REGULARよりもトーンを抑えたデザインにします。

### 3.4. アクションボタンの配置安定化
- 実績画面下部の `.modal-actions` 内に配置される `[RETURN TO TITLE]` ボタンは、開発者用デバッグリセットボタン（`#achievement-reset-btn`）の有無にかかわらず、**常に右寄せ**（`margin-left: auto;`）で美しく配置されます。

## 4. データ永続化とストレージ仕様

- **LocalStorage キー**: `burst_cascade_achievements`
- **データ構造**:
  ```json
  {
    "progress": {
      "regular": {
        "easy":   { "achievements": {}, "best": {}, "winStreak": 0, "totalWins": 0, "totalLosses": 0, "totalDraws": 0, "totalSuicideWins": 0 },
        "normal": { "achievements": {}, "best": {}, "winStreak": 0, "totalWins": 0, "totalLosses": 0, "totalDraws": 0, "totalSuicideWins": 0 },
        "hard":   { "achievements": {}, "best": {}, "winStreak": 0, "totalWins": 0, "totalLosses": 0, "totalDraws": 0, "totalSuicideWins": 0 }
      },
      "mini": {
        "easy":   { "achievements": {}, "best": {}, "winStreak": 0, "totalWins": 0, "totalLosses": 0, "totalDraws": 0, "totalSuicideWins": 0 },
        "normal": { "achievements": {}, "best": {}, "winStreak": 0, "totalWins": 0, "totalLosses": 0, "totalDraws": 0, "totalSuicideWins": 0 },
        "hard":   { "achievements": {}, "best": {}, "winStreak": 0, "totalWins": 0, "totalLosses": 0, "totalDraws": 0, "totalSuicideWins": 0 }
      }
    }
  }
  ```
  `achievements` オブジェクトには、解除済みの実績IDがキーとして登録されます（例: `"win": true`）。
  `best` オブジェクトには、解除条件判定時に記録されたその実績のベスト統計値（例: `"win": 5`）が登録されます。
