# レンダリング・演出システム

## 1. 座標系 (Unified Coordinate System)
本作は多様なデバイス環境で一貫した操作感と視覚体験を提供するため、**論理座標系**と**物理座標系**を厳密に分離している。

### 1.1. High-DPI (DPR) 対応
- `devicePixelRatio` に基づき Canvas の内部解像度を物理ピクセルに合わせつつ、描画コンテキスト自体に `scale(dpr, dpr)` を適用。
- **論理座標（CSSピクセル）**: マップレイアウト計算、当たり判定（`findHexAt`）、パーティクル座標計算、および選択状態管理（`selectedHex`）に使用。
- **物理座標（DPR補正）**: キャンバス内部解像度、フォントサイズの決定に使用。

### 1.2. PointerEvent 体系
- マウスとタッチを確実に判別し、300ms のクリック遅延を排除するため、`click` ではなく `pointerup` を主操作イベントとして使用。
- **ブラウザジェスチャーの抑制**: キャンバスに `touch-action: none` を適用し、かつ `touchmove` リスナーに `{ passive: false }` を明示設定することで、標準的なスクロールやズーム干渉を排除。
- **動的選択と確定の分離**: `isTouchMoved` フラグを導入。タッチスライド（`touchmove`）中はそのマスを「選択（Hover）」対象とするが、指を離した瞬間の `pointerup` では常に「選択（Select）」状態を確定させるにとどめ、実際の「注入（Confirm）」はそのマスへの次回の単独タップによってのみ実行される。これにより、スライド操作直後の意図しない誤爆を防止。
- **ハイライト表示の一貫性**: 共通メソッド `updateHoveredNeighbors` により、スライド操作中とタップ操作（選択時）の両方で、周囲マスのプレビュー表示が正しく更新・維持される。
- **デバイス別操作最適化**: マウス操作（PC）では 1 クリック即時確定、タッチ操作（モバイル）では 2 タップ確定（またはスライド後に 1 タップ確定）を明確に分離。

### 1.3. レイアウトの安定化
- iPad / モバイルブラウザでの表示崩れを防ぐため、オーバーレイ要素を `position: absolute` で `main` 要素内に配置し、ヘッダーとのスタッキングコンテキストの干渉を回避。

## 2. 演出システム (Dynamic FX)
### 2.1. 3D 垂直座標補正
- ヘクスが積み重なって厚みを持つため、パーティクル生成時には各ヘクスの現在の高さ（天面）を考慮した `y - height` の座標を起点とする。
- 物理的な衝撃が伴わないアクション（高さ0のタイル配置等）ではパーティクル生成を抑止し、視覚的な一貫性を保持。

### 2.2. カスケード演出
- 連鎖爆発、パワー再構築ドット、コアダメージ粒子など、全ての演出が論理座標ベースで同期して描画される。
